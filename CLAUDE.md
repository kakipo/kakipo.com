# このプロジェクトにおける特別な指示

## アクセシビリティ（最重要）

**コードを編集・追加する際は、毎回 WCAG 2.1 AA に準拠していることを確認すること。**

具体的には以下を常にチェックする:

- **1.4.3** テキストのコントラスト比 4.5:1 以上（大文字・見出しは 3:1 以上）
- **1.4.11** UI コンポーネント（ボタン・入力欄・境界線など）のコントラスト比 3:1 以上
- **2.4.1** スキップリンクを `<body>` 直下に設置
- **2.5.3** `aria-label` がボタンの表示テキストを含むこと（Label in Name）
- **3.1.2** 英語など日本語以外のコンテンツには `lang="en"` などを付与
- **4.1.3** ステータスメッセージは `aria-live` で通知
- 装飾的な要素には `aria-hidden="true"`
- インタラクティブ要素には `:focus` スタイル
- タッチターゲットは `min-height: 44px`
- `prefers-reduced-motion` を尊重

## デザイン方針

- テキストデザインは極力シンプルに
- コンテンツカードにすりガラス（frosted glass）効果を維持
- リンクは標準的なスタイル（青文字・下線）

## Git / デプロイ

- **push する前には必ずユーザーに確認すること**
- push の確認は、他の操作（ドキュメント更新など）への「ok」と兼用しない。push してよいか、を独立して明示的に問うこと
- push の確認をする前に、変更点のサマリをユーザーに伝えること

## JavaScript 方針

- **外部ライブラリは可能な限り使わない**（Vanilla JS を優先）
- コードの優先順位: **パフォーマンス** > **可読性**

## ドキュメント管理

- 作業後、`OVERVIEW.md` に追記が必要かどうかを確認する
- 追記が必要な場合はその内容をユーザーに伝え、**許可を得てからアップデートする**

## 実装メモ

- Jekyll + GitHub Pages 構成
- GoL はライブラリ不使用の Vanilla JS（`Uint8Array` ダブルバッファ、差分描画）
- `_layouts/home.html` と `preview.html` は常に同期させること
